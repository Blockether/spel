Findings

BUG-001 Critical api /login
DEV Verified
Found by: Hunter · Challenged by: Skeptic · Verdict: Referee CONFIRMED

XTDB database error exposed to end users

When login fails, the application returns a raw XTDB stack trace including database connection details, query structure, and internal Java class names. This leaks implementation details and is a security risk.

Steps to Reproduce

  1. Navigate to /login
  2. Enter invalid credentials (e.g., test@example.com / wrongpassword)
  3. Click "Sign In"
  4. Observe: raw stack trace with XTDB internals displayed in UI
Agent Analysis

The error handler does not sanitize backend exceptions before rendering. XTDB query errors propagate through Fulcro's error boundary and are rendered as-is. This pattern exposes database technology, query patterns, and internal namespaces.

Confidence: High Impact: Information disclosure — database technology and query internals exposed to attackers
BUG-002 Critical functional /dashboard
DEV Verified
Found by: Hunter · Verdict: Referee CONFIRMED

Dashboard shows incorrect plan data after subscription change

After changing subscription plans, the dashboard continues to display the old plan details until a hard refresh. Cached Fulcro state is not invalidated when backend plan data changes.

Steps to Reproduce

  1. Login as a patient with an active plan
  2. Navigate to /plans and change to a different plan
  3. Navigate back to /dashboard
  4. Observe: old plan name and features still displayed
  5. Hard refresh (Ctrl+Shift+R) to see correct data
Agent Analysis

Fulcro's client-side normalization cache is not being invalidated after the plan mutation completes. The transact! for plan change succeeds on the backend but the load! for dashboard data uses stale cache.

Confidence: High Impact: Users see wrong plan info — could lead to billing confusion and support tickets
BUG-003 High ux /login → /dashboard
Found by: Hunter · Verdict: Referee CONFIRMED

No return-URL after auth redirect

When accessing any gated page (dashboard, schedule, plans) while unauthenticated, the user is redirected to /login but the original target URL is lost. After successful login, users always land on /dashboard regardless of their original destination.

Confidence: High Impact: Users lose context and must manually navigate to their intended page after every login
BUG-004 High ux /register
Found by: Hunter · Verdict: Referee CONFIRMED

Registration form allows submission with empty required fields

The registration form can be submitted with all fields empty — no client-side validation prevents this. The server returns an unhelpful error and the user sees no field-level feedback.

Confidence: High Impact: Users get no guidance on what's wrong — leads to frustration and abandonment

Recommendations

Actionable items to improve the product, prioritized by impact.

1
Sanitize All Backend Error Responses
BUG-001 exposes XTDB internals to end users. Add a global error boundary that catches all backend exceptions and returns generic, user-safe messages. Log full stack traces server-side only. This is a security-critical fix.
2
Fix Fulcro Cache Invalidation on Mutations
BUG-002 shows stale data after plan changes. Ensure all mutations that change backend state also trigger a re-load of affected queries. Use Fulcro's :post-mutation or :refresh keys to invalidate cached data.
3
Implement Login Rate Limiting
BUG-008 enables brute-force attacks. Add server-side rate limiting (e.g., 5 attempts per minute per IP) with exponential backoff and account lockout after repeated failures.
4
Add Return-URL to Auth Redirects
BUG-003 loses user context on auth redirect. Store the original URL as a query parameter (?return=/schedule) and redirect after successful login. Standard pattern for SPA auth flows.
5
Implement Client-Side Form Validation
BUG-004 and BUG-012 show missing validation and feedback. Add Fulcro form-state validation with inline error messages, required-field indicators, and success confirmations.
6
Fix Keyboard Navigation and Accessibility
BUG-006 and BUG-011 block screen reader users. Add proper ARIA roles, alt text, and ensure all interactive elements are keyboard-focusable with visible focus indicators.
7
Disable Development Instrumentation in Production
BUG-017 leaks framework details. Ensure the production ClojureScript build uses :optimizations :advanced and removes Fulcro Inspect, development warnings, and debug logging.
